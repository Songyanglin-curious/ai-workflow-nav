# AGENTS.md

本文件定义 `server/` 后端协作规则。除非当前任务明确覆盖，否则默认遵守本规范。

## 目录规则

当前后端结构如下：

```text
server/
├─ config/
│  └─ app.config.jsonc        # 本地配置
├─ sql/
│  └─ prompts.sql             # 建表脚本
├─ src/
│  ├─ app/                    # 应用装配、公共错误
│  ├─ config/                 # 配置读取与校验
│  ├─ infra/                  # 数据库、路径、外部工具
│  └─ modules/
│     ├─ prompts/             # 提示词模块
│     └─ test/                # 测试模块
├─ db/                        # 运行时 SQLite 文件
└─ dist/                      # 构建产物
```

- 业务代码统一放在 `src/`
- 建表脚本统一放在 `sql/`
- 本地配置统一放在 `config/app.config.jsonc`
- `dist/` 是构建产物，不在这里直接改源码
- `node_modules/`、`db/runtime.sqlite` 都不属于业务实现目录

## 分层规则

- 调用顺序固定为：`routes -> service -> repo / infra`
- `routes` 只收参数、回响应，不直接读写数据库或磁盘
- `service` 负责业务流程与领域语义
- `repo` 只负责表读写，不混入正文文件处理
- `infra/db` 只处理数据库连接和建表
- `infra/paths` 只处理路径解析
- `infra/tools` 只处理外部工具调用
- 不把业务判断塞进 `infra`
- 不把业务语义下沉到基础设施层
- 不把业务流程伪装成简单 CRUD

## 接口规则

- 当前接口统一使用 `POST`
- 接口语义通过接口名称区分，不靠 HTTP 动词区分
- 新接口先补文档，再落代码
- 查询、写入、删除都保持统一风格，不要同模块内混用一套 `POST` 一套 REST

## 错误规则

- 业务错误统一走 `src/app/appError.ts`
- 公共错误码统一放在 `src/app/errorCodes.ts`
- 统一错误出口放在 `src/app/errorHandler.ts`
- 不直接把业务失败当作裸 `500`
- 查无数据、参数不合法、配置错误都应返回明确业务错误码

## 配置与路径规则

- 本地配置从 `config/app.config.jsonc` 读取
- 涉及工作区、外部项目、工具路径的能力，先加配置，再写实现
- 数据库存相对路径；绝对路径只在配置层和执行阶段出现
- 不要把本机绝对路径写死在业务代码里

## 数据与模型规则

- 真实关联统一使用 ID
- 当前提示词模型只保留：`id`、`name`、`description`、`category`、`content_file_path`、`created_at`、`updated_at`
- 当前阶段不再引入标签字段
- 如果新增字段、删除字段、调整语义，必须先同步文档和 SQL，再改实现

## 变更顺序规则

涉及模块功能、字段、接口、表结构的改动，必须从上到下逐层修改，不允许跳层直接改代码。

默认顺序如下：

1. 先改 `docs/` 里的模块设计、页面设计、接口设计
2. 再改 `docs/` 里的 SQL 设计
3. 再改 `server/sql/*.sql`
4. 最后改 `src/` 中的实现代码

- 如果是公共能力，优先先改公共模块文档，不要把公共规则塞回业务模块下面
- 如果发现当前代码和文档不一致，先以最新确认过的文档语义为准，再推进实现

## 命名规则

- 路由：`xxx.routes.ts`
- 业务：`xxx.service.ts`
- 数据访问：`xxx.repo.ts`
- 类型：`xxx.types.ts`
- 校验结构：`xxx.schemas.ts`
- 辅助逻辑：`xxx.utils.ts`

## 当前共识

- 提示词模块当前优先做主链路，不提前扩展不稳定能力
- 类别是当前保留的唯一分类字段
- 类别字典是公共能力，应独立设计、独立实现
- 排序类需求应显式落字段，不靠临时数组顺序或前端写死顺序

## 交付要求

- 优先做小而完整的改动
- 修改后应能说明：为什么这样放、为什么不放到别处
- 如果后端结构或公共规则发生稳定变化，需要同步更新本文件
