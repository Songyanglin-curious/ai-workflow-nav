# AGENTS.md

本文件定义本仓库的仓库级公共约束。除非当前任务明确覆盖，否则整个仓库默认遵守本文件。

## 作用范围

- 本文件对整个仓库生效
- `server/AGENTS.md` 补充后端局部规则
- `web/AGENTS.md` 补充前端局部规则
- 子目录规则不冲突时默认同时生效

## 核心规则

- 注释统一使用中文，只写帮助理解结构、意图、边界的内容
- 禁止防御性编程；错误应显式暴露，不允许吞错、静默 fallback、模糊返回
- 不允许为了“先跑起来”偷偷改写设计语义
- 主体逻辑前置，命名优先可读、稳定、易扫视
- 只有出现稳定复用再抽公共层，不为未来可能性提前做抽象
- 改动优先做小而完整的一步，不做无边界扩展

## 文档优先

- 涉及功能、字段、接口、表结构、模块职责的改动，优先先改 `docs/`
- 公共能力优先放在公共模块文档，不挂在某个业务模块下面
- 如果代码与文档冲突，先以当前已确认的文档语义为准

## 变更顺序

涉及模块功能、字段、接口、表结构的改动，必须从上到下逐层修改，不允许跳层直接改代码。

默认顺序：

1. 先改 `docs/` 中相关模块设计、页面设计、接口设计
2. 再改 `docs/` 中相关 SQL 设计
3. 再改实现侧的 SQL 或配置
4. 最后改实际代码

## 当前共识

- 真实关联统一使用 ID
- 路径类语义先在设计中说明，再进入实现
- 排序类需求应显式落字段，不依赖隐式顺序
- 当前提示词模块已去掉标签语义，只保留类别语义
- 当前类别字典属于公共能力，应独立设计、独立实现
