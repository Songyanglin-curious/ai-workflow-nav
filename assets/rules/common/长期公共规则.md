# 长期公共规则

这些规则适用于前后端共同的设计与实现，不绑定某一端。

## 收录规则

1. 文档优先，涉及字段、接口、表结构、模块职责变化时，先改文档再改代码。
2. 变更必须从上到下逐层修改，不允许跳层直接改实现。
3. 公共能力先独立设计，不挂到某个业务模块下面。
4. 真实关联统一使用 ID，名称只用于展示、检索和人工理解。
5. 排序需求必须显式落字段，不依赖临时顺序或前端写死顺序。
6. 先砍掉低价值复杂度，再推进主链路可用性。
7. 长期稳定、会被后续多个任务重复遵守的内容，才进入 assets。

## 暂不放入这里的内容

以下内容不要放进本文件：

- 前端组件库、样式、布局细节
- 后端接口风格、分层、错误处理细节
- 具体业务字段
- 排障步骤
- 模板和检查清单
